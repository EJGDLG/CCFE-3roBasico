# Instructivo General – Tercera Unidad
## CCFE 3ro Básico

Durante la tercera unidad se trabajarán actividades prácticas relacionadas con el uso de hojas de cálculo, creación de páginas web básicas y simulación de circuitos electrónicos.

## 1. Computación – Actividades de Excel o Google Sheets

### Tema general
Uso de hojas de cálculo para resolver ejercicios de análisis utilizando:

- Fórmulas básicas
- Funciones de Autosuma
- Análisis de datos

### Modalidad
Las actividades de Computación se pueden realizar en:

- **Microsoft Excel**, si el estudiante cuenta con el programa.
- **Google Sheets**, si no cuenta con Excel.

### Indicaciones
El estudiante deberá resolver las hojas de trabajo asignadas, aplicando las herramientas vistas en clase.

En estas actividades deberá analizar los datos y decidir qué herramienta utilizar según lo que pide cada pregunta.

Puede utilizar:

- **Fórmulas básicas**: `+`, `-`, `*`, `/`
- **Funciones de Autosuma**:
  - Suma
  - Promedio
  - Contar números
  - Máx
  - Mín

### Entrega
Las actividades deberán enviarse por correo electrónico al profesor:  
`edwinjosegabriel@gmail.com`

#### Recomendaciones para enviar el archivo
El correo debe incluir:

- Nombre completo del estudiante
- Grado y sección
- Archivo adjunto de Excel o enlace de Google Sheets
- Asunto claro, por ejemplo:  
  `Tarea Computación Tercera Unidad - Nombre del estudiante`

#### Indicaciones para el trabajo escrito - de tablas de Excel
El estudiante debe realizar las tareas en:

- Hojas tamaño oficio
- A mano
- Con letra clara
- Bien identificado

Cada hoja debe incluir:

- Nombre completo
- Grado y sección
- Fecha
- Título de la actividad


## 2. Programación – Actividades de HTML

### Tema general
Creación y modificación de páginas web básicas utilizando HTML, CSS y JavaScript.

Las actividades trabajadas incluyen ejercicios como:

- Páginas HTML básicas
- Uso de tablas
- Inserción de contenido
- Login básico con usuario y contraseña
- Modificación de páginas para negocios ficticios

### Modalidad
Las actividades de Programación deberán trabajarse y entregarse utilizando **GitHub**.

### Indicaciones
Cada estudiante deberá:

1. Crear un repositorio en GitHub para sus actividades de Programación.
2. Subir las imágenes donde se pueda vizualizar los resultados de los codigos.
3. Agregar al profesor como colaborador del repositorio.

#### Colaborador obligatorio
El estudiante debe agregar como colaborador al usuario:  
`EJGDLG`

### Entrega
La entrega será por medio de GitHub.

El repositorio debe contener:

- Evidencia de que la página funciona correctamente (imagen donde se vizualice el codigo y el resultado)

### Recomendaciones
El estudiante debe verificar que:

- Las imágenes carguen bien.
- El repositorio esté visible para el profesor.
- El profesor `EJGDLG` haya sido agregado como colaborador.

## 3. Robótica – Actividades en hojas y simulación en Wokwi

### Tema general
Uso básico de componentes electrónicos y simulación de circuitos con Arduino en Wokwi.

Las actividades incluyen:

- Uso de LED y resistencia
- Control de LED con botón
- Semáforo con tres LEDs
- Identificación de conexiones
- Uso de entradas y salidas digitales

### Parte 1: Trabajo escrito a mano
Las actividades teóricas de Robótica deberán realizarse **a mano**.

#### Indicaciones para el trabajo escrito
El estudiante debe realizar las tareas en:

- Hojas tamaño oficio
- A mano
- Con letra clara
- Bien identificado

Cada hoja debe incluir:

- Nombre completo
- Grado y sección
- Fecha
- Título de la actividad
- Desarrollo ordenado de cada ejercicio

#### Entrega física
La parte escrita deberá entregarse físicamente en clase.

### Parte 2: Simulación en Wokwi
Además del trabajo escrito, el estudiante deberá realizar las simulaciones solicitadas en [Wokwi](https://wokwi.com/).

#### Indicaciones para Wokwi
El estudiante debe:

1. Entrar a Wokwi.
2. Crear el circuito correspondiente.
3. Conectar correctamente los componentes.
4. Ejecutar la simulación.
5. Tomar capturas de pantalla como evidencia.

#### Evidencias obligatorias
Debe subir **2 capturas de pantalla**:

- Captura donde se vea el circuito armado.
- Captura donde se vea el código o la simulación funcionando.

#### Repositorio obligatorio para Robótica
Las capturas deberán subirse a un repositorio de GitHub llamado:  
`robotica3unidad`

#### Contenido del repositorio
El repositorio debe incluir:

- Capturas del circuito en Wokwi
- Capturas del código o simulación funcionando
- Nombre claro de cada imagen
